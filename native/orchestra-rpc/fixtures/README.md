# Conformance fixtures

Generated by `scripts/dump-rpc-fixtures.ts` (backend workstream A1) against a
seeded test store — one `{method, params, result}` capture per RPC method and
one `{channel, args}` capture per event channel (arrays of captures per file
are fine). `tests/conformance.rs` deserializes every capture into the typed
mirror, re-serializes, and fails on any semantic divergence
(docs/ui-rpc-protocol.md §7). With no `*.json` here the rig skips with a
notice — CI regenerates the corpus and runs it on the combined branches.
